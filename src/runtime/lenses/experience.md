# Experience Lens

## Use When

涉及 UI、CLI、API ergonomics、developer experience、状态反馈、错误恢复、
可访问性或高频操作流程时使用。

## Questions

- 用户能否看懂当前状态、可用动作与完成结果？
- 主路径是否高效，错误是否可恢复且不丢上下文？
- loading、empty、partial、blocked 与 success 状态是否完整？
- 键盘、屏幕尺寸、辅助技术和长文本下是否仍可操作？

## Evidence

提供真实交互、截图、浏览器或 CLI 输出、可访问性检查及失败路径验证；不要只用
静态代码阅读替代体验验证。
