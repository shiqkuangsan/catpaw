# Migration: 1.2.0

Runtime upgrade: 1.1.0 -> 1.2.0
Project impact: additive

## Operations

- add: `.catpaw/index.md`: introduce frontmatter block with a `runtime` field. The actual value is set by `catpaw:upgrade-project --apply` to the installed runtime target version, not literally `1.2.0`. If the file already has a frontmatter block without `runtime`, insert the field at the top of the existing block. If `runtime` already exists with a different value, overwrite to the target version on apply and report the prior value in the dry-run.

## User Decisions

(none)

## Notes

This migration introduces the project board version stamp. From 1.2.0 onward, every project `.catpaw/` board carries `runtime: x.y.z` in `index.md` frontmatter, which `catpaw:upgrade-project` reads to decide which migration deltas to replay.

For boards that predate this stamp, treat the absence of `runtime:` as `1.0.0` and replay every migration in `(1.0.0, installed-runtime]`.

The stamp value is always the installed runtime target version of the upgrade, not the migration file's own version. This keeps one-shot upgrades correct: a 1.0 → 1.5 jump stamps `runtime: 1.5.0`, never `1.2.0`; a runtime-only release with no migration still advances the stamp to show the board has been processed by that runtime.
