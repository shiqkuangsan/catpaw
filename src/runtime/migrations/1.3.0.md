# Migration: 1.3.0

Runtime upgrade: 1.2.0 -> 1.3.0
Project impact: additive

## Operations

- register: `<boardPath>`: append or upsert this board into `~/.catpaw/state/projects.json` per `specs/03-project-directory.md` §7. Resolve `boardPath` to absolute path. Use the existing `index.md` `runtime:` stamp as the entry's `stamp` value; if absent, treat as `1.0.0`. Set `registeredVia: upgrade-project`, `registeredAt: <today>` (only when entry is new), `lastSeenAt: <today>`, `lastSeenVia: upgrade-project`.

## User Decisions

(none)

## Notes

This migration is the first non-board-only delta. It writes to `~/.catpaw/state/projects.json`, not to any project `.catpaw/` files.

Replay semantics from 1.0.0 → 1.3.0 in a single `upgrade-project --apply` pass:

1. Apply 1.1.0: add graph frontmatter to plan/test-matrix/review-summary files.
2. Apply 1.2.0: add `runtime:` field to `.catpaw/index.md` frontmatter (value = target version, see 1.2.0 notes).
3. Apply 1.3.0: upsert board into the global registry.

The registry write is idempotent: running upgrade-project on an already-registered board re-applies upsert (same boardPath, refreshed `stamp` and `lastSeenAt`).

If `~/.catpaw/state/projects.json` does not exist, the first upgrade-project run creates it with `{"schemaVersion": 1, "updatedAt": <today>, "projects": []}` before appending.

Boards that never run upgrade-project (e.g., new boards created via init-project after 1.3.0 ships) are registered by `init-project` directly, not by this migration.
