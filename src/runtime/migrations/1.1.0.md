# Migration: 1.1.0

Runtime upgrade: 1.0.x -> 1.1.0
Project impact: additive

## Operations

- add: `.catpaw/plans/active/*.md`: frontmatter block with `id`, `req`, `status`, `updated`, `closed`. Default values: infer `id` from filename slug; infer `req` from filename `^([A-Z]+-\d+)` prefix or leave blank with a comment; `status: active`; `updated:` today's date; `closed: null`.
- add: `.catpaw/plans/archive/*.md`: same fields as active plans, but `status: done` (or `cancelled` if filename or content makes that clear); `closed:` last modification date if available, otherwise today's date.
- add: `.catpaw/tests/matrices/*.md`: frontmatter block with `id`, `req`, `plan`, `status`, `updated`, `closed`. Default values: infer `id` from filename (`T-` prefix or filename slug); infer `req` from filename prefix; infer `plan` from sibling plan file when an unambiguous match exists, else leave blank; `status:` derive from existing `Status:` line in the body if present, else `active`; `updated:` today; `closed:` today when status terminal, else `null`.
- add: `.catpaw/reviews/*/summary.md`: frontmatter block with `req`, `plan`, `status`, `mode`, `updated`, `closed`. Default values: infer `req` and `plan` from directory name; `status:` derive from existing `Status:` line in the body if present, else `active`; `mode:` if formal review subdirectory shape exists use `formal`, else prompt user (`light` or `formal`); `updated:` today; `closed:` today when status terminal, else `null`.

## User Decisions

- review-mode: For each `reviews/<dir>/summary.md` without a clear `mode` signal, ask user to choose `light` or `formal`. Default suggestion: `light` for L2 work, `formal` for L3 work; if level cannot be inferred, default `light`.

## Notes

This migration is purely additive. Files that already have a frontmatter block must be left untouched even if individual fields are missing — operators will be reported as findings rather than auto-filled, to avoid clobbering hand-edited frontmatter.

Any field whose value cannot be inferred safely must be left as a placeholder comment such as `# req: TODO infer from path` rather than invented.
